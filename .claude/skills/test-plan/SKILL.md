# Test Plan Skill

Use esta skill quando criar ou alterar uma feature.

## Saída esperada

- Testes unitários.
- Testes de integração.
- Testes negativos.
- Testes manuais, quando necessário.
- Dados de teste sem segredos reais.
- Critério de aceite verificável.

## Checklist de segurança

- Logs não contêm segredo.
- Permissões negadas são testadas.
- Erros não expõem stack/segredo ao usuário final.
- Ações sensíveis geram auditoria.
