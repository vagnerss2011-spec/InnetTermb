# Sprint 06 — Governança, changelog e release

Você é o agente responsável por consolidar governança profissional do monorepo.

## Leitura obrigatória

- `docs/11-devops-github-ci.md`
- `docs/23-governanca-versionamento-changelog.md`
- `VERSIONING.md`
- `CHANGELOG.md`
- `.github/PULL_REQUEST_TEMPLATE.md`
- `.github/CODEOWNERS`

## Objetivo

Criar scripts e validações para manter o projeto organizado com múltiplos agentes.

## Entregas

- Script de validação de changelog.
- Script de validação de JSON schemas.
- Atualização do CI para rodar validações.
- Checklist de release em Markdown.
- Convenção de tags por componente.

## Critério de aceite

- CI falha se contratos JSON forem inválidos.
- PR template exige changelog/ADR quando aplicável.
- Release checklist existe e é fácil de seguir.
- Versionamento por componente está documentado.
